PROJECT OVERVIEW
================================================================================
CertEye is a production-ready Flask web application for managing internal 
PKI certificates. It receives certificate data from PowerShell agents, stores 
it in a database, monitors expiration dates, calculates risk levels, and 
displays everything on a clean Bootstrap dashboard.

DATABASE SCHEMA
================================================================================
Table: certificates

Columns:
  • id (Integer PK)              - Unique identifier
  • server_name (String)         - Server hostname [indexed]
  • subject (String)             - Certificate subject CN
  • issuer (String)              - Certificate issuer CA
  • thumbprint (String, Unique)  - Certificate fingerprint [indexed]
  • expiry_date (DateTime)       - Expiration timestamp [indexed]
  • certificate_template (String)- Certificate type (WebServer, DC, etc)
  • created_at (DateTime)        - Record creation timestamp
  • updated_at (DateTime)        - Last update timestamp

Calculated Properties:
  • days_to_expiry               - Automatically calculated
  • risk_level                   - Automatically classified
  • is_expired                   - Boolean flag
  • expiry_status                - Human-readable status

================================================================================
API ENDPOINTS
================================================================================

1. POST /api/internal-certificates
   Purpose: Accept certificate data from PowerShell agent
   Body: JSON with server_name, subject, issuer, thumbprint, expiry_date
   Response: 201 Created | 200 OK (update) | 400 Bad Request | 500 Error

2. GET /api/certificates
   Purpose: Retrieve all certificates
   Query params: risk_level, sort_by (expiry_date|server_name|risk_level), order
   Response: 200 OK with array of certificates

3. GET /api/alerts
   Purpose: Get certificates expiring within threshold
   Query params: days (default 30), sort_by
   Response: 200 OK with expiring certificate list

4. GET /api/certificate/<id>
   Purpose: Get single certificate details
   Response: 200 OK | 404 Not Found

5. DELETE /api/certificate/<id>
   Purpose: Delete certificate from database
   Response: 200 OK | 404 Not Found

6. GET /api/health
   Purpose: API health check
   Response: 200 OK with status and timestamp

7. GET /api/dashboard-stats
   Purpose: Get statistics for dashboard
   Response: 200 OK with counts and metrics

================================================================================
RISK LEVEL CLASSIFICATION
================================================================================

EXPIRED (Red):    < 0 days (already expired)
CRITICAL (Red):  ≤ 15 days to expiry (immediate action required)
HIGH (Orange):   16-30 days to expiry (plan remediation)
NORMAL (Green):  > 30 days to expiry (routine management)

POWERSHELL INTEGRATION
================================================================================

Complete PowerShell scripts provided in POWERSHELL_EXAMPLES.md:

1. Simple Certificate Send:
   $cert = Get-ChildItem -Path Cert:\LocalMachine\My | Select -First 1
   $payload = @{
     server_name = $env:COMPUTERNAME
     subject = $cert.Subject
     issuer = $cert.Issuer
     thumbprint = $cert.Thumbprint
     expiry_date = $cert.NotAfter.ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ssZ")
     certificate_template = "WebServer"
   } | ConvertTo-Json
   
   Invoke-WebRequest -Uri "http://localhost:5000/api/internal-certificates" `
       -Method POST -ContentType "application/json" -Body $payload

2. Batch Processing:
   Loop through certificate store and send all certificates

3. Scheduled Task:
   Run daily via Windows Task Scheduler

PRODUCTION CONSIDERATIONS
================================================================================

SECURITY:
  • All dates stored in UTC
  • Unique thumbprint constraint prevents duplicates
  • Input validation on all endpoints
  • Error handling prevents information leakage
  • No authentication yet (recommend Phase 2)

PERFORMANCE:
  • Indexed searches on common fields
  • Efficient risk calculation (property methods)
  • Stateless API design (scalable)

DEPLOYMENT:
  Recommended upgrades:
  1. Use PostgreSQL instead of SQLite
  2. Run behind nginx/Apache reverse proxy
  3. Enable HTTPS/SSL certificates
  4. Add authentication (API keys or JWT)
  5. Use gunicorn/uWSGI instead of Flask dev server
  6. Set secure SECRET_KEY in config.py
  7. Configure automated database backups
  8. Monitor disk space
